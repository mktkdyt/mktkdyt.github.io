ストレージアカウント追加手順

January 14, 2021

概要

ストレージアカウントを追加することでストレージアカウント経由で Azure の仮想マシンとデータの受け渡しが可能になります。

手順

Azure のポータル画面で [ストレージアカウント] をクリックし [＋追加] をクリックします。

下記内容を入力し [確認と作成] をクリックします。 ${ResourceGroupName} にはリソースグループを，${SubscriptionName} にはサブスクリプションを指定します。 ${resourcegroupname} には ${ResourceGroupName} をすべて小文字にした文字列を指定します。

[基本] で下記内容を入力します。

項目 設定
サブスクリプション ${SubscriptionName}
リソースグループ ${ResourceGroupName}
ストレージアカウント名 ${resourcegroupname}
場所 東日本
パフォーマンス Standard
アカウントの種類 StorageV2 (汎用 v2)
レプリケーション 読み取りアクセス地理冗長ストレージ (RA-GRS)

[ネットワーク] で下記内容を入力します。

項目 設定
接続方法 パブリックエンドポイント (すべてのネットワーク)
ルーティングの優先順位 Microsoft ネットワークルーティング

[データ保護] で下記内容を入力します。

項目 設定
コンテナーのポイントインタイムリストアを有効にする 無効
BLOB の論理的な削除を有効にする 無効
コンテナーの論理的な削除を有効にする 無効
ファイル共有の論理的な削除を有効にする 無効
BLOB のバージョン管理を有効にする 無効
BLOB の変更フィールドを有効にする 無効

[詳細] で下記内容を入力します。

項目 設定
安全な転送が必須 有効
TLS の最小バージョン バージョン 1.2
BLOB パブリックアクセスを許可する 有効
BLOB のアクセスレベル ホット
階層構造の名前空間 無効
大きいファイルの共有 無効
カスタマーマネージドキーのサポート 無効

[タグ] は何も指定せず，[確認と作成] をクリックします。

[作成] をクリックします。
